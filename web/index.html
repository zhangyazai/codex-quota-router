    路由规则

    规则应用于所有当前可参与路由的账号；该状态不代表模型请求已实时验证。

    Responses 会话会优先粘在原账号；额度耗尽、认证失败或限流导致切号时，路由器会从内存回放完整上下文。状态不落盘，进程重启、空闲超过 24 小时、缓存淘汰、单条 transcript 超过 32 MiB，或请求引用账号绑定的 conversation、item/file ID 时，不能保证无感恢复。

    Codex 用户级配置

    复制后写入对应系统的用户级配置文件：

    macOS：~/.codex/config.toml

    Windows：%USERPROFILE%\.codex\config.toml

    Linux：~/.codex/config.toml

      Token 内容不会在列表中显示。复制配置会按需读取指定 Token；删除或轮换后，旧 Token 立即失效。

      Codex OAuth 采用与 CLIProxyAPI 类似的 OpenAI 浏览器登录流程，由服务端完成 PKCE、回调和凭据保存，再把已授权账号接入 Codex Responses 上游；不会读取本机 auth.json 或浏览器 Cookie，也不会在页面回显 OAuth 凭据。

      公网访问与管理安全

      服务仍只监听 127.0.0.1；公网访问必须通过同机 HTTPS 反向代理，并保留原始 Host。

      只能填写 HTTPS Origin，不包含路径、查询参数或片段。

      至少 12 个字节；启用公网访问前必须设置，留空保留已保存密码。

      反向代理必须启用 TLS、限制 Host，并覆盖单值 X-Forwarded-For 与 X-Forwarded-Proto=https。启用公网访问后，本机管理也需要密码；不要把 4000 端口直接映射到公网。
      所有修改一次保存
      Key 留空会保留服务器中已保存的值。
      服务仅监听本机地址；公网访问必须经过配置的 HTTPS 反向代理。账号 Key 与余额认证秘密不会在页面中回显。
